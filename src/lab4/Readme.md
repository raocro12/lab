# Задание 2. Параметризации. Маски типов и их ограничения.
## Задача3. Начало отсчета
### Текст задачи
Создайте метод, принимающий Коробку из задачи 3.1.1, и кладет в неё трехмерную точку
координат (из задачи 2.1.5) с произвольными значениями. Метод должен позволять передавать
Коробку с более чем одним видом параметризации.  
(Обобщенная коробка.
Создайте сущность Коробка, которая обладает следующими характеристиками:  
- Может хранить один произвольный объект в один момент времени.
- Объект можно получить и разместить на хранение в любой момент времени.
- Если объект забирают из коробки – ссылку на этот объект необходимо обнулить.
- Если объект кладут в коробку, но она не пуста – необходимо выкинуть исключение.
- Имеет метод проверки на заполненность.
- Методы класса должны работать с тем типом данных, который был указан во время
создания объекта)  
### Алгоритм решения
Создадим обобщённый класс `public class Box<T>` так как коробка должна иметь воможность хранить данные любого типа.
Класс имеет одно поле `private T item`, метод для размещения объекта  `public void put(T newItem)` и метод для извлечения `public T take()`, метод для проверки на заполненость `public boolean isEmpty()`. 
Метод, метод который принимает коробку и кладет в неё трёхмерную точку `public static void putPoint(Box<? super Point3D> box)`. Метод может принимать только коробку типа `Point3D` и всех её родительских классов. По условию задачи метод всегда кладёт в коробку трёхмерную точку с произвольными знаениями, то есть можем не передавать в метод точку, а сразу в методе положить в коробку 3 мерную точку с рандомными значениями.
В главном методе создаём экземпляр коробки `Box<Point3D> box = new Box<>();` и кладём точку `PointInBox.putPoint(box);`.

Коробка содержит: {0.6230685906414989; 0.7318637273670089; 0.7839593893008984}
